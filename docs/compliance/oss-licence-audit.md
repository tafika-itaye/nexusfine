# Open-Source Licence Audit

**Status:** v1.0 · **Owner:** Engineering + Compliance
**Cadence:** quarterly + on every new package addition
**Purpose:** ensure NexusFine carries no incompatible open-source obligations
into government procurement.

---

## 1. Policy

| Licence class | Action |
|---|---|
| MIT, BSD-2, BSD-3, Apache 2.0, ISC, MS-PL, MS-RL | ✅ Approved. Add freely. |
| LGPL 2.1 / 3.0 (dynamic-linking only) | 🟡 Approved with attribution + LICENSES file entry. |
| MPL 2.0, EPL 2.0 | 🟡 Approved with attribution; isolate to its own assembly. |
| GPL 2.0, GPL 3.0, AGPL 3.0 | ⛔ Blocked. Replace before merge to main. |
| Proprietary / commercial-only | ⛔ Blocked unless a per-project licence agreement is in force and filed under `docs/compliance/third-party-licences/`. |
| Unknown / no licence file | ⛔ Blocked. Treat as proprietary. |

## 2. Audit procedure

### 2.1 Generate the package list

```powershell
cd C:\Users\HP1\Documents\GitHub\nexusfine
# Direct + transitive packages for the whole solution
Get-ChildItem -Recurse -Filter *.csproj | ForEach-Object {
    Write-Host "=== $($_.Name) ==="
    dotnet list $_.FullName package --include-transitive
}
```

### 2.2 Resolve licences

Use the helper script:

```powershell
.\scripts\audit-oss-licences.ps1 -OutFile docs\compliance\oss-licence-report.csv
```

This walks every `.csproj`, queries the NuGet API for each unique package,
extracts the licence expression and the project URL, and writes a CSV with:
`project, packageId, version, licenceExpression, licenceUrl, projectUrl, status`.

### 2.3 Review & sign

- Open the CSV in Excel.
- Sort by `status` — anything not "Approved" gets a row in §4 below with
  an owner + a "remediate by" date.
- Sign the report (paper signature + scan, OR DocuSign).
- File the signed report under `docs/compliance/third-party-licences/oss-{YYYY-MM-DD}.csv`.

### 2.4 Block PRs that add unknown licences

A CI check (planned, Wave 4) runs the same audit on every PR and fails the
build if a non-approved licence appears in the diff.

## 5. Attribution file

A consolidated `NOTICE.md` ships at the repository root listing every
third-party package with its copyright line and a link to the original
licence text. This satisfies the attribution clauses in MIT, BSD, and
Apache-2.0.

The `NOTICE.md` is regenerated by the audit script and committed as part of
the quarterly review.
